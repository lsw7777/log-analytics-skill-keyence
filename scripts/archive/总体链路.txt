 结论：运行 .\run-all.ps1 时，本项目不会调用 OpenCode 中的大模型，也不会调用 OpenAI/LLM API。

  大模型是否被调用
  没有。项目脚本里没有发现 openai、chat/completion、model、prompt、OpenCode agent 调用等逻辑。

  容易误解的网络/API调用有 3 类，但都不是大模型：

  - Azure 登录：azure_log_query.ps1:163
    Connect-AzAccount @connectParams

  - Log Analytics 查询：azure_log_query.ps1:223
    Invoke-AzOperationalInsightsQuery @queryParams

  - License Graph API：analyze.ps1:338
    Invoke-RestMethod ... /subscribedSkus

  另有 Microsoft Service Tags 下载函数：

  - log-analyzer-core.ps1:489
  - log-analyzer-core.ps1:492

  这是下载微软 IP 段 JSON，也不是大模型调用。

  是否使用了 SKILL.md
  运行 .\run-all.ps1 时不会使用 SKILL.md。

  SKILL.md 只是给 OpenCode/Codex 这类代理看的技能说明文件，用于指导“人机协作时怎么操作这个项目”。PowerShell 脚本没有读取它。项目中没有类似：

  Get-Content SKILL.md

  或任何加载 SKILL.md 的逻辑。

  简短判断：这个项目当前是 PowerShell + Azure Monitor Log Analytics + Microsoft Graph 的数据查询/分析脚本，不是 LLM 分析脚本。OpenCode/大模型只可能在你让我修改代码、解释代码时由外部开发代理使用；run-all.ps1 本身不会调用大模型。
