# Log Analytics Skill 项目架构分析

## 一、项目概述

这是一个用于分析 **Azure Log Analytics（微软日志分析服务）** 中 Office 365 审计日志的工具集。项目运行在 **Azure 中国云** 环境，主要目标是帮助运维和安全团队查询、分析和可视化 Office 365 的审计日志数据。

## 二、通俗理解项目是做什么的

想象一下，你们公司用了微软的 Office 365（包括 Power BI、SharePoint、Exchange 等），所有用户的操作都会被记录下来——比如谁看了什么报表、谁导出了数据、谁在凌晨3点访问了敏感文件。这些日志被存到了 Azure Log Analytics 这个"日志仓库"里。

这个项目的作用就是：
1. **从日志仓库里取数据** — 通过 PowerShell 脚本连接到 Azure，把日志查出来
2. **分析数据** — 统计谁最活跃、哪些操作最常见、有没有可疑行为
3. **生成报告** — 自动做出漂亮的 HTML 报告，有图表、有表格，还能中英文切换

## 四、技术架构详解

### 4.1 数据获取层 — `azure_log_query.ps1`

这是整个项目的**唯一数据源**，负责：

1. **认证登录** — 使用 `Connect-AzAccount` 通过浏览器交互式登录到 Azure 中国云
2. **执行查询** — 调用 `Invoke-AzOperationalInsightsQuery` 向 Log Analytics 发送 KQL 查询
3. **导出结果** — 支持将查询结果导出为 CSV 文件

**关键配置：**
- 云环境：AzureChinaCloud（中国云）
- Workspace ID：`703a5771-97fc-4bf3-a585-f607d18c4479`
- Tenant ID：`420c4dab-8603-402f-afe0-75bc28c51c13`

### 4.2 数据处理层 — PowerShell 分析脚本

查询到的数据通过 PowerShell 进行：
- 字段统计（用户数、操作数、工作负载分布）
- 风险分析（失败操作、可疑IP、非工作时间活动）
- 数据聚合（Top N 排行、时间线）

### 4.3 展示层 — HTML 报告生成

生成**自包含的 HTML 文件**（所有 CSS/JS 都内嵌），提供：
- 可视化图表（柱状图、环形图、时间线）
- 交互式表格（排序、分页、搜索）
- 多语言切换（中文 ↔ 日本語）
- 风险分析面板

## 八、关键技术点

### KQL 查询语言
项目使用 Kusto Query Language (KQL) 查询日志，例如：
```kql
-- 查看最新1条记录（探测字段）
AuditGeneralDCR_CL | take 1

-- 按时间范围查询
AuditGeneralDCR_CL | where TimeGenerated >= datetime(2026-05-07) | sort by TimeGenerated desc

-- 按工作负载过滤
AuditGeneralDCR_CL | where Workload == 'PowerBI' | sort by TimeGenerated desc
```

### 数据暂存策略
为避免重复查询 Azure（耗时且可能断连），项目采用**先导出、后分析**的策略：
1. 查询结果导出到 `~\AppData\Local\Temp\opencode\General_YYYYMMDD.csv`
2. 后续所有分析都从本地 CSV 读取
3. 支持大数据集分片处理

### 自包含 HTML
生成的报告是单个 `.html` 文件，无需任何外部依赖：
- CSS 内嵌在 `<style>` 标签中
- JS 内嵌在 `<script>` 标签中
- 图表用纯 CSS/SVG 实现，不依赖任何图表库

## 九、总结

本项目是一个**轻量级、自包含**的日志分析工具集，通过 PowerShell 脚本与 Azure Log Analytics 交互，将 Office 365 审计日志转化为可视化的 HTML 报告。当前仅支持 `AuditGeneralDCR_CL` 一张表，但架构设计清晰，易于扩展到其他日志表。
