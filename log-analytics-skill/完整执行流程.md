# Log Analytics 完整执行流程

## 一键执行（推荐）

### 使用 run-all.ps1

这是最简单的执行方式，一条命令完成所有步骤：

**重要：不要复制 ``` 符号！``` 只是 Markdown 文档的格式标记，不是命令的一部分。**

**正确执行方式（逐行复制，不要包含 ```）：**

```
cd c:\AB_Projects\OpenCode效率化\log-analytics-skill\log-analytics-skill\skills-prj\bulk-log-analyzer
```

```
powershell -ExecutionPolicy Bypass -File .\run-all.ps1
```

**指定表名：**

```
powershell -ExecutionPolicy Bypass -File .\run-all.ps1 -TableName "SharePointAuditDCR_CL"
```

**指定时间范围（过去48小时）：**

```
powershell -ExecutionPolicy Bypass -File .\run-all.ps1 -Hours 48
```

**强制重新登录：**

```
powershell -ExecutionPolicy Bypass -File .\run-all.ps1 -ForceLogin
```

### 执行流程

```
[1/4] 获取日志数据
        ↓
    调用 azure_log_query.ps1
    自动打开浏览器进行 Azure 认证
    数据导出到 CSV 文件
        ↓
[2/4] 加载并统计数据
        ↓
    计算: 总记录数、唯一用户、唯一操作、
          工作负载分布、成功/失败比率
        ↓
[3/4] 生成HTML报告
        ↓
    调用 analyze.ps1
    生成自包含的 HTML 文件
        ↓
[4/4] 在浏览器中打开
        ↓
    自动打开默认浏览器显示报告
```

### 输出文件位置

| 文件类型 | 路径 |
|---------|------|
| CSV 数据 | `%USERPROFILE%\AppData\Local\Temp\opencode\{表名}_{日期时间}.csv` |
| HTML 报告 | `%USERPROFILE%\AppData\Local\Temp\opencode\report_{日期时间}.html` |

### HTML 网址格式

HTML 报告是**本地文件**，网址格式为：

```
file:///C:/Users/{用户名}/AppData/Local/Temp/opencode/report_{日期时间}.html
```

例如：
```
file:///C:/Users/D9352/AppData/Local/Temp/opencode/report_20260522_091500.html
```

**注意：** 这不是 HTTP 服务器，而是直接打开本地 HTML 文件。

## 分步执行

### 步骤 1: 获取日志数据

```powershell
cd c:\AB_Projects\OpenCode效率化\log-analytics-skill\log-analytics-skill\skills-prj\bulk-log-analyzer

# 探测字段（取1条记录）
powershell -ExecutionPolicy Bypass -File .\azure_log_query.ps1 -Query "AuditGeneralDCR_CL | take 1" -Hours 24

# 查询并导出到 CSV
powershell -ExecutionPolicy Bypass -File .\azure_log_query.ps1 `
    -TableName "AuditGeneralDCR_CL" `
    -Query "AuditGeneralDCR_CL | sort by TimeGenerated desc" `
    -Hours 24 `
    -ExportCsv "$env:USERPROFILE\AppData\Local\Temp\opencode\General_$(Get-Date -Format 'yyyyMMdd_HHmm').csv"
```

### 步骤 2: 统计数据

```powershell
# 检查 CSV 行数
powershell -Command "(Import-Csv -Path '$env:USERPROFILE\AppData\Local\Temp\opencode\General_20260522.csv' -Encoding UTF8).Count"
```

### 步骤 3: 生成 HTML 报告

```powershell
powershell -ExecutionPolicy Bypass -File .\analyze.ps1 `
    -CsvPath "$env:USERPROFILE\AppData\Local\Temp\opencode\General_20260522.csv" `
    -OutputPath "$env:USERPROFILE\AppData\Local\Temp\opencode\report_20260522.html" `
    -AnalysisDate "2026-05-22"
```

### 步骤 4: 在浏览器中打开

```powershell
# 在默认浏览器中打开
Start-Process "$env:USERPROFILE\AppData\Local\Temp\opencode\report_20260522.html"
```

## HTML 报告内容

生成的 HTML 报告包含以下内容：

1. **概览卡片**
   - 总事件数
   - 唯一用户
   - 唯一操作
   - 工作负载数量
   - 成功/失败计数

2. **可视化图表**
   - 活动时间线（按小时）
   - 工作负载分布（环形图）
   - 活跃用户排行（柱状图）
   - 操作类型排行（柱状图）
   - 客户端 IP 排行（柱状图）
   - 成功/失败比率（分段条）

3. **风险分析**
   - 失败操作
   - 非工作时间活动
   - 高权限操作
   - 敏感数据事件
   - 可疑 IP
   - IP 多用户关联
   - 服务账户活动

4. **详细数据表**
   - 分页显示（每页50行）
   - 可排序列
   - 操作名称悬停提示（中日文解释）

5. **术语表**
   - 操作名称中日文对照
   - 可折叠显示

6. **多语言支持**
   - 中文/日文切换
   - 所有 UI 文本支持 i18n

## 常见问题

### Q: 如何查看 HTML 网址？
A: 脚本执行完成后会在控制台显示完整的 file:// 网址。也可以直接在文件资源管理器中双击 HTML 文件。

### Q: HTML 网址可以在其他电脑访问吗？
A: 不可以。这是本地文件，只能在本机访问。如需共享，可以将 HTML 文件复制到其他位置或通过 Web 服务器托管。

### Q: 如何共享报告？
A: 直接发送 HTML 文件即可，它是自包含的，无需其他依赖。接收方用浏览器打开即可查看。

### Q: 认证失败怎么办？
A: 运行 `.\run-all.ps1 -ForceLogin` 强制重新登录。确认账号有 Log Analytics 工作区的读取权限。

### Q: 查询返回空结果？
A: 检查：
1. `-Hours` 参数是否覆盖了数据时间范围
2. 表中是否有数据（先用 `| take 1` 测试）
3. KQL 查询条件是否正确
