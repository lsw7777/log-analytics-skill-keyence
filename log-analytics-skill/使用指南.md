# Log Analytics Skill 使用指南

## 一、项目启动

### 1. 环境准备

**必需软件：**
- **PowerShell 7+** (推荐) 或 Windows PowerShell 5.1+
- **Azure PowerShell 模块** (脚本会自动安装)
  - `Az.Accounts`
  - `Az.OperationalInsights`
- **Azure 账号** — 需要对 Log Analytics 工作区有读取权限

**可选软件：**
- **VS Code** — 编辑脚本和查看 HTML 报告
- **Chrome/Edge 浏览器** — 查看生成的 HTML 报告

### 2. 首次运行

打开 PowerShell，进入项目目录：

```powershell
cd c:\AB_Projects\OpenCode效率化\log-analytics-skill\log-analytics-skill\skills-prj\bulk-log-analyzer
```

首次运行会自动安装 Azure 模块并提示浏览器登录：

```powershell
.\azure_log_query.ps1 -Query "AuditGeneralDCR_CL | take 1" -Hours 24
```

**登录流程：**
1. 脚本会自动打开浏览器
2. 使用微软账号登录（Azure 中国云）
3. 授权 Azure PowerShell 访问
4. 回到 PowerShell 继续执行

**如果遇到 401/403 错误：**
```powershell
.\azure_log_query.ps1 -ForceLogin    # 重新登录
.\azure_log_query.ps1 -UseDeviceCode # 使用设备代码流（无浏览器环境）
```

## 二、使用 azure_log_query.ps1 查询日志

### 基本用法

```powershell
# 1. 快速探测字段结构（取1条记录）
.\azure_log_query.ps1 -Query "AuditGeneralDCR_CL | take 1" -Hours 24

# 2. 查询最近 N 小时的数据
.\azure_log_query.ps1 -Query "AuditGeneralDCR_CL | sort by TimeGenerated desc" -Hours 24

# 3. 指定表名查询
.\azure_log_query.ps1 -TableName "SharePointAuditDCR_CL" -Query "SharePointAuditDCR_CL | take 1" -Hours 48

# 4. 导出到 CSV 文件
.\azure_log_query.ps1 -TableName "AuditGeneralDCR_CL" -Query "AuditGeneralDCR_CL | sort by TimeGenerated desc" -Hours 24 -ExportCsv ".\General_20260521.csv"
```

### 常用参数

| 参数 | 说明 | 示例 |
|-----|------|------|
| `-Query` | KQL 查询语句 | `-Query "AuditGeneralDCR_CL \| take 1"` |
| `-TableName` | 表名（默认 AuditGeneralDCR_CL） | `-TableName "SharePointAuditDCR_CL"` |
| `-Hours` | 查询过去 N 小时（默认 1） | `-Hours 24` |
| `-ExportCsv` | 导出 CSV 文件路径 | `-ExportCsv ".\output.csv"` |
| `-ForceLogin` | 强制重新登录 | `-ForceLogin` |
| `-UseDeviceCode` | 使用设备代码流 | `-UseDeviceCode` |

### 支持的日志表

| 表名 | 简称 | 说明 |
|-----|------|------|
| `AuditGeneralDCR_CL` | General | Office 365 通用审计日志 |
| `SharePointAuditDCR_CL` | SPAudit | SharePoint 审计日志 |
| `MessageTraceDataDCR_CL` | MsgTrace | 邮件追踪数据 |
| `AssignedLicensesDCR_CL` | Licenses | 已分配许可证信息 |
| `AzureADUsersDCR_CL` | AADUsers | Azure AD 用户信息 |
| `MailboxStatisticsDCR_CL` | Mailbox | 邮箱统计信息 |
| `WQCLogDCR_CL` | WQC | WQC 日志 |

### KQL 查询示例

```kql
-- 探测字段
AuditGeneralDCR_CL | take 1

-- 按时间排序
AuditGeneralDCR_CL | sort by TimeGenerated desc

-- 按用户过滤
AuditGeneralDCR_CL | where UserId contains "user@domain.com" | sort by TimeGenerated desc

-- 按工作负载过滤
AuditGeneralDCR_CL | where Workload == "PowerBI" | sort by TimeGenerated desc

-- 失败的操作
AuditGeneralDCR_CL | where IsSuccess == false | sort by TimeGenerated desc

-- 特定时间范围
AuditGeneralDCR_CL | where TimeGenerated >= datetime(2026-05-21) | sort by TimeGenerated desc

-- SharePoint 审计
SharePointAuditDCR_CL | where SiteUrl contains "sites/finance" | sort by TimeGenerated desc

-- 邮件追踪
MessageTraceDataDCR_CL | where Status == "Failed" | sort by TimeGenerated desc
```

## 三、使用 SKILL.md

SKILL.md 是 AI (OpenCode/Cline) 的操作指南，定义了完整的工作流程。

### 文件位置

- `skills-prj/bulk-log-analyzer/SKILL.md` — 批量日志分析器
- `skills-prj/interactive-query-tool/SKILL.md` — 交互式查询工具

### 工作流程

#### 批量日志分析器 (bulk-log-analyzer)

1. **探测架构** — 运行 `| take 1` 获取字段结构
2. **导出数据** — 使用 `-ExportCsv` 导出到临时 CSV 文件
3. **分析数据** — 从 CSV 文件读取并计算统计信息
4. **生成报告** — 创建自包含的 HTML 报告
5. **验证输出** — 检查报告正确性

#### 交互式查询工具 (interactive-query-tool)

1. **探测架构** — 运行 `| take 1` 获取字段结构
2. **获取数据** — 使用 KQL 条件过滤查询
3. **构建 HTML** — 创建交互式查看器（搜索、排序、分页）
4. **验证输出** — 测试各项功能

## 六、常见问题

### Q: 如何安装 Azure PowerShell 模块？
A: 脚本会自动检测并安装。如需手动安装：
```powershell
Install-Module -Name Az.Accounts -Scope CurrentUser -Force
Install-Module -Name Az.OperationalInsights -Scope CurrentUser -Force
```

### Q: 登录失败或 401 错误？
A: 运行 `.\azure_log_query.ps1 -ForceLogin` 重新登录。确认账号有 Log Analytics 读取权限。

### Q: 查询返回空结果？
A: 检查：
1. `-Hours` 参数是否覆盖了数据时间范围
2. KQL 查询条件是否正确
3. 表中是否有数据（先用 `| take 1` 测试）

### Q: CSV 文件太大怎么办？
A: 在 KQL 中添加过滤条件减少数据量：
```kql
AuditGeneralDCR_CL | where Workload == "PowerBI" | where TimeGenerated >= datetime(2026-05-21) | sort by TimeGenerated desc
```

### Q: 如何查看生成的 HTML 报告？
A: 直接用浏览器打开：
```powershell
start chrome .\report.html
